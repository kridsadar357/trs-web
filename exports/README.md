# Deployment export bundle

This folder is generated by **`scripts/export-for-vps.ps1`** (or a manual `mysqldump`).

| File | Purpose |
|------|--------|
| `trs_web_full_dump.sql` | Full MySQL dump of database `trs_web` (schema + data). |
| `public_media.zip` | `public/hero` and `public/uploads` (extract into project `public/` on the server). |

## Restore on VPS (MySQL in Docker example)

```bash
# Copy files to server, then:
docker exec -i YOUR_MYSQL_CONTAINER mysql -u root -pYOUR_PASSWORD trs_web < trs_web_full_dump.sql
```

Or from host with `mysql` client:

```bash
mysql -h 127.0.0.1 -u root -p trs_web < trs_web_full_dump.sql
```

## Restore media

```bash
cd /path/to/trs-web
unzip -o public_media.zip -d public_tmp
# Ensure hero/ and uploads/ end up under public/
mv public_tmp/hero public/ 2>/dev/null || true
mv public_tmp/uploads public/ 2>/dev/null || true
```

On Windows, extract `public_media.zip` so `hero` and `uploads` sit under `public/`.

## Security

- Do **not** commit `.sql` dumps to a **public** GitHub repository (passwords, PII, session data).
- Use a **private** repo, or store backups only on the VPS / object storage.
